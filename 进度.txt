## 任务6：文件基础 CRUD + 列表展示

### 一、任务内容说明

当前文件模块有骨架代码，但存在以下问题需要修复：

| 问题 | 现状 | 需改成 |
|------|------|--------|
| userId 来源 | Controller 从请求参数拿 `userId` | 从 Sa-Token 当前登录用户（`UserContext`）获取 |
| 返回格式 | 直接返回 `List` / `String` | 统一用 `Result` 包装 |
| 权限校验 | 无，任何人传 fileId 就能操作 | AOP 切面：操作前校验文件是否属于当前用户 |
| 删除逻辑 | 只改 `fileStatus=2` | 同时写入 `recycle_bin` 表，记录删除时间 |
| 同名检查 | 无 | 创建/重命名/移动时检查目标位置是否已存在同名文件 |
| 分页 | 无 | 文件列表接口加分页参数 |
| 批量删除 | 无 Controller | 新增 `batchDelete` |

### 二、为什么用 AOP 做权限校验？

文件模块有 7 个操作都涉及单文件归属校验（重命名、移动、复制、删除、下载、预览等）。如果每个方法里都写：

```java
FileInfo f = getById(fileId);
if (!f.getUserId().equals(currentUserId)) throw new BusinessException("无权操作");
```

代码重复 7 遍。用 AOP 切面可以在方法执行前统一拦截，只写一次。

核心思路：
- 自定义注解 `@CheckFileOwner`
- 切面类解析方法参数中的 `fileId`，查数据库比对 `userId`
- 不匹配直接抛异常

这和 Sa-Token 的路由拦截是两层防护：路由拦截管"登没登录"，AOP 拦"是不是你的文件"。
